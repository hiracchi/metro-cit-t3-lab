## レポート課題5（6〜7日目）：AI連携Webサイトの構築とシステムの完成

### 【基礎知識】ファイアウォールとシステム連携
これまでに構築したWebサーバーとAI APIを連携させ、独自のAIチャットシステムを完成させる。完成したシステムを安全に運用するために、ファイアウォール（ufw）を用いて不要な通信ポートを閉じることが不可欠である。

### 1. 目的
WebサーバーとAI APIを連携させ、独自のAIチャットシステムを完成させる。

### 2. 実験方法（使用機器と手順）

1. フロントエンドとバックエンドの構築
`index.html` に入力フォームを作成し（Webフロントエンド）、PHPやPython(Flask)等を用い、Webフォームの入力をOllama APIに転送し、結果を表示する「AIチャット画面」を構築する（バックエンド連携）。

2. セキュリティ設定

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 53/udp
sudo ufw enable
```

`ufw` で必要なポート（22, 80, 53）以外を閉じる。

### 3. 結果と考察
ブラウザから `http://www.jikken.internal` にアクセスし、AIと会話できるか確認する。

📷 **【エビデンス取得】**
1. ポート制限の確認結果。
2. ブラウザ上でAIと正常に会話が行えている画面。
以上2点のキャプチャを取得し、システムが正しく連携・保護されていることを考察すること。

### 4. 課題
1. このシステムを学外に広く公開する場合、どのようなセキュリティリスクが考えられるか？
2. ファイアウォール設定にて、ポート22（SSH）、80（HTTP）、53（DNS）のみを許可した。もしポート22を開けたまま学外に公開した場合、どのような危険性があるか。また、それを防ぐために2日目で設定した対策について説明せよ。
3. 構築したネットワークサーバーを運用し続ける場合、OSやソフトウェアのアップデート（`update`コマンド等）を定期的に行う必要がある。不具合を放置してアップデートを怠った場合に生じる問題点や法的な責任について考察せよ。
