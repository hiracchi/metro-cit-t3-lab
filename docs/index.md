## 注意事項

本演習では、コンピュータネットワーク上のクライアントにサービスを提供するネットワークサーバーの構築を行う。
各自のWindows 11クライアント端末からリモートでアクセスし、Proxmox仮想演習サーバー上にUbuntuの仮想マシンをセットアップして構築作業を実施する。

#### 1. レポートの作成と提出
本演習では、計7日間（1日180分）の演習を通じて、5つのレポートを作成・提出する。学生が書く実験レポートは、見た目の綺麗さよりも内容の正確さや他人に理解できることが大切である。
したがって、(1)どのような実験を行い (2)どの結果が得られ (3)その結果からどのようなことが考えられるか、明確にわかるように書く必要がある。

レポートはA4サイズの用紙を用い、以下の構成で作成すること。

1. **目的**: 何のための実験か。
2. **実験方法（使用機器を含む）**: 使用した環境（Windows 11クライアント、Proxmoxサーバー、Ubuntu等）やソフトウェア名を含め、実施した手順と実行した主要なコマンドを記述する。
3. **結果と考察**: 取得した画面キャプチャ（エビデンス）や測定値等の結果と、結果に対する自分の考え、理論値とのずれやエラーが発生した場合の仮説と検証、改善点などをまとめて記述する。
4. **課題**: 各章末に設定された課題への回答。
5. **参考文献**: 参考にした書籍やWebサイトのURL等を記述する。リストアップするだけでなく、**必ず本文中の参考にした箇所で参考文献番号（など）を用いて参照すること**。

#### 2. エビデンス（画面キャプチャ）の取得と図表のルールについて
実験結果を証明するために、各手順の完了時に画面のスクリーンショット（エビデンス）を取得する。
* **取得方法**: ターミナルでコマンドの実行結果が表示されている状態で画面をコピーし、レポートに貼り付けること。
* **図表の表記ルール**: グラフやキャプチャ画像などの図番号とタイトルは図の下に、表の表番号とタイトルは表の上に表示すること。
* **本文中での参照と説明**: 図や表を貼り付けるだけでなく、**必ずレポート文章中で図番号や表番号（「図1に示すように～」「表1は～」など）を用いて参照すること**。その上で、図表が示している内容を具体的に明記すること（「見ればわかる」は不可である）。
