## レポート課題2（2日目）：ネットワーク設定とSSH鍵認証

### 【基礎知識】IPアドレスとネットワークの基本
情報機器をネットワークに参加させるためには、ホスト名、IPアドレスとネットマスク、デフォルトゲートウェイのパラメータが必要である。
IPアドレスのうち、全世界で通信可能なものをグローバルアドレス、組織内（LAN）で利用されるものをプライベートアドレスと呼ぶ。

### 1. 目的
CUIでのネットワーク設定（Netplan）を習得し、安全なリモート管理環境を構築する。

### 2. 実験方法（使用機器と手順）

1. 設定ファイルのバックアップ

```bash
sudo cp /etc/netplan/00-installer-config.yaml /etc/netplan/00-installer-config.yaml.bak
```

上記コマンドで設定ファイルをバックアップする。

2. 固定IP設定
`vi` や `nano` エディタで設定ファイルを編集し、指定されたIPアドレス、ゲートウェイ、DNS（8.8.8.8等）を設定する。

3. SSH鍵認証の設定

3-1. Windows 11のPowerShellで `ssh-keygen` を実行する。
3-2. 公開鍵をサーバーへ転送する（`ssh-copy-id` または手動貼り付け）。
3-3. サーバーの `/etc/ssh/sshd_config` を編集し、`PasswordAuthentication no` に設定する。

### 3. 結果と考察
設定を反映させるコマンドを実行する。

```bash
sudo netplan apply
```

📷 **【エビデンス取得】** Windowsからパスワードなしでログインできるか確認し、その成功画面をキャプチャして説明を加えること。

### 4. 課題
1. パスワード認証を禁止し、鍵認証のみに制限するセキュリティ上の理由は何か？
2. 情報機器をネットワークに参加させるために必要なパラメータのうち「デフォルトゲートウェイ」はどのような役割を持つか、内部（LAN内）通信と外部通信の違いに触れて説明せよ。
3. ネットワーク設定の変更前にファイルのバックアップ（`cp`コマンドでのコピー）を作成した。システム設定ファイルを編集する前にバックアップを取ることがなぜ重要なのか、考察せよ。
